Validate .rst Document Links With rst-url-validator

The posts on this blog are written in reStructuredtext format. I recently had an idea to check my broken urls for broken tags or urls that are no valid. So I did what any other programmer does in 2023 and opened up an AI assistant. Bing quickly generated the structure of the Python script, but like usual it required some tinkering and refinement. You can find the Python script in the rst-url-validator Github repo.

Install Python Library Dependencies

The script uses requests to validate the link and rich to print color-coded output to the terminal screen.

pip install requests
pip install rich

Clone The Github Repo

git clone git@github.com:erickbytes/rst-url-validator.git

Install Python Library Dependencies

python3.12 rst-url-validator.py your_file.rst

It parses each url tag, checks for required characters and sends a HEAD request to the url to check if it still loads successfully. Some websites return status codes like 403 (permission denied), 301 (redirect), or 404 (page not found). If the status code returned is 200, it's safe to say the page loaded.

This script helped me find the broken links, refresh or clean them up tons on this blog. If you're working with .rst documents, give it a try!
